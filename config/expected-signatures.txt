# The only signing certs a downloaded stock APK may carry. download-apk.sh's
# verify() accepts a stock APK only if one of its signing-cert SHA-256 digests is
# listed here for its package, so a tampered or repackaged build from an
# untrusted mirror is rejected before it reaches the patcher.
#
# Format: <package>  <cert-sha256-hex-lowercase>   (# comments, blank lines ok)
#
# The value is apksigner's "certificate SHA-256 digest", the same value morphe
# and anddea ship in each package's signature list. Google rotates these keys
# (APK Signature Scheme v3), so apksigner can print a different signer cert per
# minSdk band; both of a package's certs are Google's, so both are listed.
#
# To add a package, run `apksigner verify --print-certs <stock.apk>`, take every
# certificate SHA-256 digest, and cross-check each against the vendor's signature
# list before committing.
